NAMIBIA ID

Applying for a government ID in Namibia; the easy way.

As a project for the User Interface Construction course (CS-E5220) in Aalto University , we were developing a solution that would digitalize the process of applying for a government ID in Namibia.

My Roles

Research and discover the application phases to complete the procedure of applying for an id.
Create a Command Line Interface app, to allow the users to apply for an id.
Create part of the Graphical User Interface app, for the same purpose as above.
Report the procedures and techniques applied.

Challenge

The main purpose of the course and of the project was to digitalize the process of applying for a government ID in Namibia. After the research phase, that every group member conducted independently, we discovered that the process involved visiting some government offices multiple times, other than filling in various documents. Thus, we decided to try to optimize the process itself, by reducing the number of unnecessary trips and thus the time needed to complete the procedure.

Approach

Individual part

Initially, every member of the group was tasked to create an application with a basic command line interface; the objective was to study the flow of the process and begin the digitalization process.
The process I decided to digitalize was the one to report the birth or the death of a family member.

Using NodeJS, the commander, and the inquirer packages I created an interactive command line interface, with colors, detailed help, and checkboxes that guides step by step the users in the procedure; this simplified mode is meant for inexperienced users, that prefer to be guided throughout the whole process and receive more feedback along the way. Alternatively, for more advanced users, it is possible to input in the command line a single-line command, like in the traditional ones, to complete the entire process; in this case, there is just a final confirmation that the process is terminated.

Besides the CLI application, I also created a prototype for smartphones; the prototype, built with Mockplus had the same features of the CLI interface. As the CLI, the prototype exposed two modes of operation; a guided one, this time with buttons and text-boxes and a power user one that had one single text box to insert the whole command.

The individual report I wrote contains additional details about the features, the design decisions, and the guidelines applied.

Team part

Later in the course, I teamed up with my two classmates Vojta and Mario to prototype and build a more detailed application, this time with a Graphical User Interface. Since Mario and Vojta both chose as the topic for the initial app the application for a government ID, we decided to keep that as the topic for the rest of the projects.
Thus, we developed a web application using Angular and Firebase; the application was developed only for smartphones, so if you are on desktop, I suggest to toggle on the device mode. (on Google Chrome press the F12 key, then click on Toggle device toolbar)
Additional details regarding the technologies, guidelines, and design decisions are in the group report.

Techniques

We focused on applying most of the guidelines available for User Interfaces, such as:

  • Nielsen’s 10 Usability Heuristics.
  • Schneiderman’s 8 Golden Rules.
  • Smith & Mosier.
  • W.C.A.G. 2.

Results

By the end of the course, we built a functioning web app* that digitalizes and optimizes the application process for an ID; besides, it applies the guidelines studied in the course and other ones as well.
* As mentioned before, I suggest to turn on the device mode if you are on desktop.
