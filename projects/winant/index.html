WinANT PRO

Modern cross-platform applications for network operators.

Starting in the Spring of 2018, I worked as a Software Developer Intern at Monkeycube S.R.L. a software company based in Milan, Italy. Together with one of their developers, we developed a modern cross-platform application, based on NW Js, React Native, and Semantic UI. The application was a porting of a very old program written in Visual Basic; its purpose was, given a set of data and variables, to compute the optimal position for antenna towers for radio/television operators.

My Roles

Re-implemented the whole UI using Semantic UI.
Translated some of the old features from Visual Basic to React.
Discussed and implemented additional features, like a pdf report generator.
Studied and proposed architecture and software design mplementations.
Participated to the meetings with the final customer.

Challenge

The main task was to create a porting of the old Visual Basic application, that was able to run only on a Virtual Machine equipped with Windows 98 or 2000. Additionally, the final customer asked for a cross-platform version, that could run not only on Windows 10, but at least on the latest version of Mac OS as well.
Finally, the GUI of the application needed to be updated to respect the latest standard and to make it more user-friendly and intuitive.

Approach

When I started my internship in the company, the basic structure of the application along with some features had already been implemented in React Native. For this reason, I directly jumped in the development of the most important features requested by the customer maintaining the structure already in place. After a first month that I used to get familiar with the architecture and workflow, I started developing the first features.
Using a MS Excel files, we kept track of all the features that the customer was requesting us and each of us took care of some of them. We opted for the MS Excel solution, instead, for example, of GitHub issues, because we wanted to have something that was easy to share with the final customer and to edit for them.
After the first batch of features was ready and shipped to the customer (compiled into Windows binaries), I suggested to improve the GUI of the product, to polish it and make it more user-friendly. So, after some researches, we opted for Semantic UI because it was easily integrable into React Native, as opposed to, for example, the classic bootstrap that requires JQuery and thus is not really meant to be used with React.
Semantic UI allowed us to build a responsive GUI, adapt for display on different devices and screen sizes, and to easily apply themes to improve the look and feel of the interface.

Finally, after around three month of studying the platform and implementing basic features, I started to take care of bigger issues, such as design and implement brand new functions. By the end of the internship, I implemented a pdf report generator, that collected some basic data from the GUI and from the user to then generate a pdf report with only the field needed. Additionally, it exported various graphs generated by the app.

Results

By the end of my internship the app had almost all the features requested by the customer and could already be used in production.
